Modular vs Tightly Coupled Components

Demonstrating why separation of concerns leads to better maintainability and reusability

❌ The Problem: Tightly Coupled Components

Monolithic Approach

tmyl-button.js (500+ lines) ├── HTML template inside JS ├── CSS styles embedded ├── JavaScript logic mixed ├── Icon definitions inline └── Event handling coupled

❌ Problems:

  • Tight coupling - can't reuse CSS elsewhere
  • Large bundle size - everything loads together
  • Hard to maintain - changes affect multiple concerns
  • Poor reusability - can't use styles without JS
  • Testing complexity - must test everything together

Modular Approach

atoms/button/ ├── styles/tmyl-button.css ├── icons/button-icons.js ├── templates/button-template.js ├── controllers/button-controller.js └── index.js (orchestrator)

✅ Benefits:

  • Loose coupling - each file has single responsibility
  • Tree shaking - load only what you need
  • Easy maintenance - change styles without touching JS
  • High reusability - use CSS classes anywhere
  • Testable - test each concern independently

✅ Solution: Separated Concerns

1. Pure CSS Approach (Most Modular)

Use CSS classes directly in HTML - no JavaScript required for basic styling

<button class="tmyl-button tmyl-button--primary tmyl-button--md"> Primary Button </button> <button class="tmyl-button tmyl-button--secondary tmyl-button--sm"> Secondary </button> <button class="tmyl-button tmyl-button--ghost tmyl-button--lg tmyl-button--loading"> Loading... </button>

2. Factory Pattern Approach

JavaScript factory creates buttons using separated modules

// Modular JavaScript Factory import { ButtonFactory } from './button-system.js'; const button = ButtonFactory.create({ variant: 'primary', size: 'md', text: 'Click me', icon: 'search', onClick: () => console.log('Clicked!') }); document.body.appendChild(button);

3. Custom Element Approach

Web component that uses modular parts internally

Custom Element With Icon Loading State
<!-- Declarative HTML --> <tmyl-button-modular variant="primary" icon="search"> Search </tmyl-button-modular> <script> // Still uses modular parts internally import './tmyl-button-vanilla.js'; </script>

🎯 Modular Benefits in Action

Reuse CSS Classes Anywhere

The same CSS classes work on any HTML element - not just custom components

Link as Button
Div as Button
Span as Button

Mix and Match Icons

Icons are separated so they can be used independently

Search icon used separately

Easy Customization

/* Easy customization with CSS variables */ .custom-button { --tmyl-primary-600: linear-gradient(45deg, #ff6b6b, #4ecdc4); --tmyl-primary-700: linear-gradient(45deg, #ff5252, #26a69a); border-radius: 25px; text-transform: uppercase; }

📊 Bundle Size Comparison

Monolithic Component

tmyl-button.js: 45KB
├── HTML template: 8KB
├── CSS styles: 15KB
├── JavaScript: 12KB
├── Icons: 8KB
└── Framework: 2KB

❌ Must load everything even for simple styling

Modular System

CSS only: 8KB
CSS + Icons: 12KB
CSS + Factory: 15KB
Full component: 25KB

✅ Load only what you need with tree shaking
